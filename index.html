Space Invaders

A custom Python recreation of the classic arcade game by Averi Wylie and Shahyar Anfaz, expanded with 8-directional flight, dynamic enemy pressure, timed power-ups, accounts, profiles, friends, and SQLite-backed leaderboards.

Built Beyond the Arcade

Classic survival shooting is the core, but the project adds persistent identity, social scoring, and progression systems around each run.

Directional Flight

Held-key input maps to eight movement states, each with a matching spaceship sprite so the ship faces the direction it moves.

Scaling Invaders

Enemy pressure increases as kill thresholds are crossed, raising the number of active invaders chasing the player.

Projectile Combat

Shooting cooldowns, piercing ammo, score tracking, kill counts, and a 999,999-point win condition keep every run measurable.

Timed Power-Ups

Collectible tokens apply one active 30-second effect at a time, with the current power-up reflected in the HUD.

Profiles and History

Registered players can view personal bests, kill totals, and full score history after each saved run.

Friends Leaderboard

Add friends by username, compare their personal bests, and keep anonymous local scores separate from account-based runs.

Game systems

Power-Ups, Scores, and Local Persistence

Scores are stored in assets/scores.db, created automatically on first run. The global leaderboard aggregates registered user scores on the same machine, while local play stores anonymous scores only for that installation.

User Flow

  • Register, log in, or play locally without an account.
  • Save scores with kills, date, time, and local/account state.
  • Browse global top 1000, local scores, profiles, and friends-only rankings.
  • Add friends by username and compare best runs.

Getting Started

Install Python 3.9 or newer with Pygame, then launch the main application. No database setup is required.

Terminal Python 3.9+ 
git clone https://github.com/AveriWylie/Project-1---Space-Invaders.git
cd Project-1---Space-Invaders
pip install pygame
python application.py

Database Schema

  • users stores IDs, usernames, and SHA-256 password hashes.
  • scores stores user, score, kills, date, time, and local-play state.
  • friendships connects players for friends-only leaderboards.
